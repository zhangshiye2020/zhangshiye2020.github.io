过拟合

发表于 2020-11-21 更新于 2021-03-28 分类于 神经网络

概念

回归问题还是分类问题，主要区别在于最终目标不同，个人理解为所用的损失函数不同

MSE

回归问题常用

softmax

  • 激活函数

具有放大功能，概率大的更大，概率小的更小

cross_entropy

熵越大，信息越少

entropy:
$$
H(p) = -\sum_i P(i)\log{P(i)}
$$

cross_entropy:
$$
H(p,q) = -\sum_i p(i)\log{q(x)} = H(p) + D_{KL}(p|q)
$$

P=Q: H(p) = H(p,q)

P!=Q: 对于 one-hot 编码时，H(p) = 0

数据集划分

  • train:
    训练数据集
  • validation:
    验证数据集，防止出现过拟合
  • test:
    防止将validation作为训练集，test一般不给，但如果数据集只分为两部分，就不区分validation和test数据集

过拟合与欠拟合

欠拟合

模型复杂度小于实际模型复杂度

  • 训练精度比较低
  • 测试精度也很低

过拟合

模型复杂度超过实际模型复杂度

  • 训练精度很高
  • 测试精度很低

减少过拟合

交叉验证

将总数据集划分为k份，每次epoch使用第i份作为validation数据集，性能提升较小

Regularization

原有损失函数添加范数，迫使高阶参数权重被迫趋近于0，有些地方weight_decay表示，在没有过拟合的时候最好不要用weight_decay，否则性能下降

L1-regularization
$$
J(\theta) = -\frac{1}{m}\sum_{i=1}^m [y_i\ln\check{y}_i + (1-y_i)\ln(1-\check{y}i)] + \lambda\sum{i=1}^n|\theta_i|
$$

L2-regularization

$$
J(W;X,y) + \frac{1}{2}\lambda\cdot|W|^2
$$

动量与学习率衰减

动量

没有添加动量：只考虑当前梯度方向，往当前梯度下降方向迭代

添加动量：考虑当前梯度下降方向并同时考虑上次迭代方向，只有最原始的SGD需要手动设置momentum

learn decay

训练过程必要时刻减少学习率，比如在长时间精度到达饱和就让其缩减为一办，或者每隔多少个epoch就缩减一次

dropout,early stop

early stop

在发现test acc.衰减时，就停止训练

dropout

和regularization一样，不需要使用全部参数，每个连接都有一定概率丢掉

激活函数及其梯度

发表于 2020-11-15 更新于 2021-03-28 分类于 神经网络

概念

导数

  • 标量
  • 代表函数某个方向的变化量

偏微分

  • 对指定方向（自变量的方向）的变化量
  • 导数一种方向

梯度

  • 向量
  • 梯度长度反应该方向的增长速率
  • 梯度方向代表增长的方向

通过梯度找最小值

原理

优化算法SGD：
$$
\theta_{t+1} = \theta_{t} - \alpha\nabla(\theta_{t})
$$

  • $\alpha$ 代表学习率
  • 等到$\theta_{t}$偏微分为0的时候就是最小值

但这种方法存在一定问题，求得的算法最小值并不一定是全局最小值，可能只是局部最小值或者鞍点。当梯度到达偏微分为0的点，这时候梯度就不在下降

初始值

  • 初始值不同，找到的极值点可能也就不同

学习率

一般学习率设置的都比较小，学习率如果设置比较大，会出现梯度在极值点附近反复震荡的情况，致使不经梯度不能靠近极值点，反而会远离极值点

逃脱一个局部最小值

  • 添加一个动量
    相当于一个惯性，这样在遇到局部最小值的时候也会因为已存在的惯性冲出局部最小值点

常见函数的梯度

  • 略

激活函数

神经网络引入非线性函数

tanh函数

  • sigmod经过平移缩放
  • 会存在梯度弥散
  • 梯度更大

    sigmod 函数

  • 可能会导致梯度弥散
    $$
    \sigma(x) = \frac{1}{1+e^{-x}}
    $$

    ReLU函数

  • x>0 不会梯度弥散
    $$
    \sigma(x) = \begin{cases}
    x\qquad if\quad x > 0\
    0\qquad if\quad x < 0
    \end{cases}
    $$

leaky ReLU

  • 不会存在梯度弥散，即使在x<0

$$
\sigma(x) = \begin{cases}
x\qquad if\quad x > 0\
\alpha*x \qquad if\quad x < 0
\end{cases}
$$

损失函数

MSE

$$
loss = norm\left(y-(xw+b))\right)^{2}
$$

Softmax

$$
\sigma(z){j} = \frac{\exp(z_j)}{\sum{k=1}^K e^{z_k}}
$$

Pytorch进阶

发表于 2020-11-15 更新于 2021-03-28 分类于 神经网络

维度变换

view()/reshape()

  • view是pytorch0.3以前的默认api，reshape 是为了和numpy保持一致，功能完全一致
  • 函数需要保证矩阵变换前后元素个数不变
  • 会丢失原来的维度信息

squeeze(index)

  • 如果不给index,会能挤压的都挤压，直接挤压到一维
  • 只能挤压shape=1的维度，否则不变

unsqueeze(index)

  • 将一个维度展开，在第index插入一个维度
  • -1代表最后一个位置，-2，-3等同理

expand

  • 只在有需要的时候复制数据，一般省略复制数据过程（推荐）
  • 只改变理解方式，没添加数据
  • 传入维度信息
  • dim必须一致
  • shape为1可以扩展，否则不能扩展
  • 维度保持不变写-1

repeat

  • 实际添加数据，把数据复制了一遍
  • 效果和上面完全等效
  • 传入参数为每个维度需要复制的次数

t

  • 只适用二维
  • 转置

transpose

  • 适用于任何维度
  • 输入要交换的维度
  • 转置后数据内存上不再连续，使用contigous()函数使其变得连续

permute

  • 交换维度
  • 传入维度次序
  • 同样使用contigous

矩阵运算

add/sub/mul/div

  • 同+,-,*,/符号

matmul

  • 矩阵相乘
  • 同@
  • 只取最后两维作为运算，前面保持不变
  • 前面两维看能否是用boardcast，如果不能用boardcast就报错

mm

  • 矩阵相乘
  • 只适用于2d

power

  • 次方操作
  • 每个元素次方操作
  • 同**符号

exp/log

  • e指数/底数

floor/ceil/round/trunc/frac

  • floor向下取整
  • ceil向上取整
  • round四舍五入
  • trunc裁剪整数
  • frac裁剪小数

clamp(input,min,max,output=None)->Tensor

超出范围限制的给相应上下限

$$f(x)=
\begin{cases}
min,\quad if\quad x < min\
x,\qquad \ if\quad min < x < max\
max,\quad if\quad x > max
\end{cases}
$$

  • input: 输入矩阵
  • min: 范围下限
  • max: 范围下限

Manjaro下安装vivado问题

发表于 2020-11-11 更新于 2021-03-28 分类于 Linux

在给Manjaro安装个vivado安装时进度条卡死在Generating installed device list，运行vivado软件可以会给出缺少libtinfo5的提示。在Manjaro上这个库现已经升级到了libtinfo6，网上有人给出的解决方案是安装libtinfo5，但我这实际情况是压根找不着这两个包！

后来从stackflow找到的解决方案，没必要安装libtinfo5，libtinfo6是libtinfo5的更高版本，兼容libtinfo5，直接用libtinfo6创建libtinfo5的软链接，搞定~
